Pembukaan: “Data Tidak Hidup di Pulau Terpencil”

Di Modul 3, kita telah membedah satu dataset (Titanic) dan berhasil mengekstrak informasi darinya. Namun, di dunia nyata, data jarang sekali datang dalam satu file yang rapi. Data transaksi penjualan terpisah dari data detail produk, dan data detail pelanggan tersimpan di tabel yang lain lagi.

Tugas seorang analis data adalah menyatukan kepingan-kepingan puzzle ini untuk melihat gambaran besarnya. Di modul ini, kita akan naik level ke skenario multi-tabel yang realistis. Kita akan belajar cara menjadi “arsitek data”: menggabungkan, mengelompokkan, dan meringkas data dari berbagai sumber untuk menghasilkan wawasan yang lebih dalam. Kita juga akan berkenalan dengan SQL, bahasa universal untuk berkomunikasi dengan database.

Bagian 1: Penggabungan Data (Merging & Joining)

Kita akan langsung masuk ke studi kasus utama kita untuk modul ini, yaitu menganalisis data penjualan dari “Maven’s Restaurant”.

Skenario & Dataset

Manajemen restoran memberikan kita tiga file data terpisah: * transactions.csv: Berisi setiap item yang terjual. * products.csv: Berisi detail setiap produk, termasuk kategori dan harganya. * stores.csv: Berisi detail setiap cabang restoran.

Tantangan: transactions.csv hanya berisi ID (product_id, store_id), bukan nama produk atau lokasi toko. Untuk melakukan analisis yang berarti, kita harus menggabungkan ketiga file ini.

Unduh Dataset Restaurant Sales (Scroll untuk mencari dataset “Restaurant Sales” di halaman tersebut)

1.1. Memuat Tiga Sumber Data

Langkah pertama adalah memuat ketiga file .csv tersebut ke dalam tiga DataFrame Pandas yang terpisah.

python
import pandas as pd

# Memuat tiga file data
df_trans = pd.read_csv('transactions.csv')
df_prod = pd.read_csv('products.csv')
df_stores = pd.read_csv('stores.csv')

# Inspeksi singkat
print("--- Transaksi ---")
print(df_trans.head(3))
print("\n--- Produk ---")
print(df_prod.head(3))
print("\n--- Toko ---")
print(df_stores.head(3))

1.2. pd.merge() - Menggabungkan Dua Meja

Kita akan menggabungkan tabel ini satu per satu. Pertama, mari gabungkan df_trans dengan df_prod menggunakan kunci relasional product_id.

python
# Menggabungkan transaksi dengan detail produk
df_merged = pd.merge(
    left=df_trans,      # Tabel kiri
    right=df_prod,      # Tabel kanan
    on='product_id',    # Kunci penggabungan
    how='left'          # Tipe join: pertahankan semua transaksi
)

print(df_merged.head())

Perhatikan bagaimana kolom product_category dan price sekarang sudah ada di df_merged. Sekarang, kita gabungkan hasil ini dengan data toko.

python
# Menggabungkan hasil di atas dengan detail toko
df_full = pd.merge(
    left=df_merged,
    right=df_stores,
    on='store_id',
    how='left'
)

# Ciptakan kolom Revenue untuk analisis nanti
df_full['Revenue'] = df_full['quantity'] * df_full['price']

print("--- DataFrame Lengkap (Top 5 Rows) ---")
print(df_full.head())

Kita sekarang memiliki satu DataFrame utama (df_full) yang berisi semua informasi yang kita butuhkan.

Bagian 2: Agregasi Data Tingkat Lanjut dengan .groupby()

Dengan data yang sudah lengkap, kita bisa mulai menjawab pertanyaan bisnis yang lebih kompleks menggunakan .groupby(). Ingat filosofi Split-Apply-Combine.

2.1. Agregasi Multi-Level

Pertanyaan: “Berapa total pendapatan (Revenue) di setiap kota, dipecah lagi per kategori produk?”

python
# Mengelompokkan berdasarkan dua kolom: 'city' dan 'product_category'
pendapatan_per_lokasi_kategori = df_full.groupby(['city', 'product_category'])['Revenue'].sum()

print(pendapatan_per_lokasi_kategori)

Ini memberi kita ringkasan yang sangat detail tentang performa setiap kategori di setiap pasar.

2.2. Fungsi Agregasi Kustom dengan .agg()

Pertanyaan: “Untuk setiap kategori produk, berapa total pendapatannya, rata-rata harganya, dan total unit yang terjual?”

python
# Menerapkan beberapa fungsi agregasi sekaligus
summary_per_kategori = df_full.groupby('product_category').agg(
    Total_Pendapatan=('Revenue', 'sum'),
    Rata_Rata_Harga=('price', 'mean'),
    Total_Unit_Terjual=('quantity', 'sum')
)

# Mengurutkan hasilnya berdasarkan pendapatan tertinggi
summary_per_kategori = summary_per_kategori.sort_values(by='Total_Pendapatan', ascending=False)

print(summary_per_kategori.round(2))

Bagian 3: Berpikir dalam Dua Bahasa - Pengenalan SQL

SQL adalah bahasa standar database. Hebatnya, semua operasi yang kita lakukan di Pandas memiliki padanan di SQL.

3.1. Playground SQL di dalam Python

Kita akan memuat tiga tabel asli kita ke dalam sebuah database sementara menggunakan sqlite3.

python
import sqlite3

conn = sqlite3.connect(':memory:')

# Muat DataFrame ke database
df_trans.to_sql('transactions', conn, index=False)
df_prod.to_sql('products', conn, index=False)
df_stores.to_sql('stores', conn, index=False)

def run_query(query):
    return pd.read_sql_query(query, conn)

Latihan Utama: “Membangun Laporan Penjualan Komprehensif”

Kini giliran Anda untuk menjadi arsitek data. Gunakan DataFrame df_full yang sudah kita gabungkan.

Tugas-tugas: 1. Buatlah tabel ringkasan yang menampilkan, untuk setiap store_location: * Total Pendapatan (Revenue). * Jumlah Transaksi Unik (transaction_id). * Rata-rata Kuantitas per Transaksi (quantity). 2. (Bonus) Tulis satu query SQL yang menghasilkan tabel ringkasan yang sama persis seperti di tugas 1.

Kunci Jawaban (Klik untuk Membuka) 
python
# 1. Agregasi dengan Pandas
report_pandas = df_full.groupby('store_location').agg(
    Total_Pendapatan=('Revenue', 'sum'),
    Jumlah_Transaksi_Unik=('transaction_id', 'nunique'), # nunique untuk menghitung nilai unik
    Rata_Rata_Kuantitas=('quantity', 'mean')
).round(2).sort_values(by='Total_Pendapatan', ascending=False)

print("--- Laporan via Pandas ---\n", report_pandas)

# 2. Bonus dengan SQL
query = """
SELECT
    s.store_location,
    SUM(t.quantity * p.price) AS Total_Pendapatan,
    COUNT(DISTINCT t.transaction_id) AS Jumlah_Transaksi_Unik,
    AVG(t.quantity) AS Rata_Rata_Kuantitas
FROM
    transactions t
JOIN
    products p ON t.product_id = p.product_id
JOIN
    stores s ON t.store_id = s.store_id
GROUP BY
    s.store_location
ORDER BY
    Total_Pendapatan DESC;
"""
report_sql = run_query(query).round(2)
print("\n--- Laporan via SQL ---\n", report_sql.set_index('store_location'))
