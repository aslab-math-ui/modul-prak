Pembukaan: “Data Tidak Hidup di Pulau Terpencil”

Di dunia nyata, data jarang sekali datang dalam satu file .csv yang rapi dan lengkap. Data penumpang terpisah dari data tiket, data penjualan terpisah dari data detail pelanggan, dan data produk tersimpan di tabel yang lain lagi.

Tugas seorang analis data adalah menyatukan kepingan-kepingan puzzle ini untuk melihat gambaran besarnya. Di modul ini, kita akan belajar cara menjadi “arsitek data”: menggabungkan, mengelompokkan, dan meringkas data dari berbagai sumber untuk menghasilkan wawasan yang lebih dalam. Kita juga akan berkenalan dengan SQL, bahasa universal untuk berkomunikasi dengan database.

Bagian 1: Penggabungan Data (Merging & Joining)

Kita akan memulai dengan studi kasus baru yang melibatkan beberapa tabel.

Skenario & Dataset Baru

Sebuah toko online memberikan kita dua file data:

  • pelanggan.csv: Berisi informasi demografi pelanggan.

    customer_id customer_name city
    C01 Budi Jakarta
    C02 Citra Bandung

  • pesanan.csv: Berisi catatan transaksi.

    order_id customer_id product quantity
    101 C01 Laptop 1
    102 C03 Mouse 2

Perhatikan bahwa di pesanan.csv, kita hanya punya customer_id, bukan nama atau kota pelanggan. Untuk mengetahui pelanggan mana yang memesan Mouse, kita harus menggabungkan kedua tabel ini.

1.1. Konsep Kunci Relasional

Penggabungan tabel dimungkinkan karena adanya kunci relasional, yaitu kolom yang sama yang ada di kedua tabel. Dalam kasus kita, kolom customer_id adalah kuncinya.

1.2. pd.merge() - Pisau Bedah Penggabungan

Pandas menyediakan fungsi pd.merge() untuk melakukan penggabungan ini. Ada beberapa tipe “join” yang bisa kita lakukan.

python
import pandas as pd

# Membuat DataFrame dari data di atas
pelanggan = pd.DataFrame({
    'customer_id': ['C01', 'C02', 'C04'],
    'customer_name': ['Budi', 'Citra', 'Dedi'],
    'city': ['Jakarta', 'Bandung', 'Surabaya']
})

pesanan = pd.DataFrame({
    'order_id': [101, 102, 103],
    'customer_id': ['C01', 'C03', 'C04'],
    'product': ['Laptop', 'Mouse', 'Keyboard']
})

# --- Inner Join ---
# Hanya menggabungkan baris yang customer_id-nya ada di KEDUA tabel.
# Pelanggan 'Citra' (C02) dan pesanan 'Mouse' (C03) akan hilang.
inner_join_df = pd.merge(pelanggan, pesanan, on='customer_id', how='inner')
print("--- Inner Join ---\n", inner_join_df)


# --- Left Join ---
# Mempertahankan SEMUA baris dari tabel kiri (pelanggan), 
# dan mencocokkannya dengan data dari tabel kanan (pesanan).
# Pelanggan 'Citra' (C02) akan tetap ada, tetapi data pesanannya akan NaN.
left_join_df = pd.merge(pelanggan, pesanan, on='customer_id', how='left')
print("\n--- Left Join ---\n", left_join_df)

Inner Join berguna untuk analisis “siapa yang melakukan apa”, sedangkan Left Join berguna untuk melihat “siapa yang melakukan DAN tidak melakukan apa-apa”.

Bagian 2: Agregasi Data Tingkat Lanjut dengan .groupby()

.groupby() adalah salah satu metode paling kuat di Pandas untuk analisis. Ia bekerja berdasarkan filosofi Split-Apply-Combine:

  1. Split: Data dipecah menjadi beberapa grup berdasarkan kategori.
  2. Apply: Sebuah fungsi (seperti sum(), mean(), count()) diterapkan pada setiap grup.
  3. Combine: Hasil dari setiap grup digabungkan kembali menjadi satu data frame ringkasan.

Bagian 3: Berpikir dalam Dua Bahasa - Pengenalan SQL

SQL (Structured Query Language) adalah bahasa universal untuk berinteraksi dengan database. Sebagai seorang analis, Anda akan sering bertemu dengan SQL. Konsepnya sama persis dengan yang sudah kita pelajari di Pandas, hanya sintaksnya yang berbeda.

3.1. Playground SQL di dalam Python

Kita bisa mempraktikkan SQL langsung dari Python menggunakan library sqlite3.

python
import sqlite3

# 1. Buat koneksi ke database sementara di memori
conn = sqlite3.connect(':memory:')

# 2. Muat DataFrame Pandas kita ke dalam database sebagai tabel SQL
pelanggan.to_sql('tabel_pelanggan', conn, index=False)
pesanan.to_sql('tabel_pesanan', conn, index=False)

# 3. Fungsi untuk menjalankan query dan menampilkannya sebagai DataFrame
def run_query(query):
    return pd.read_sql_query(query, conn)

3.2. Penerjemahan Pandas ke SQL

  • Memilih Kolom (df[['col1', 'col2']]):

    sql
    SELECT customer_name, city FROM tabel_pelanggan;
    python
    print(run_query("SELECT customer_name, city FROM tabel_pelanggan;"))

  • Memfilter Baris (df[df['city'] == 'Jakarta']):

    sql
    SELECT * FROM tabel_pelanggan WHERE city = 'Jakarta';

  • Inner Join (pd.merge(..., how='inner')):

    sql
    SELECT *
FROM tabel_pelanggan
JOIN tabel_pesanan ON tabel_pelanggan.customer_id = tabel_pesanan.customer_id;

  • Agregasi (df.groupby(...).sum()):

    sql
    SELECT city, SUM(quantity)
FROM merged_table -- (Asumsikan sudah di-join)
GROUP BY city;
