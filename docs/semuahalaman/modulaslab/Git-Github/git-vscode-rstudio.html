üêô Panduan Git di RStudio & VSCode

Langkah demi langkah penggunaan Git di Windows lewat terminal RStudio (CMD) dan terminal VSCode. Lengkap dengan best practice dan link resmi tutorial.
Keywords

git rstudio windows, git vscode windows, source control rstudio, git terminal vs code, panduan git asisten lab

1 üîß Persiapan Awalan

Sebelum masuk ke IDE, pastikan:

  • Git sudah terpasang (Git Bash, atau installer resmi di Windows)
  • VSCode terinstal: https://code.visualstudio.com/
  • RStudio terinstal: https://posit.co/download/rstudio

Jika belum, ikuti panduan instalasi Git Bash di Windows pada modul sebelumnya.

2 1Ô∏è‚É£ Git di RStudio (Windows + CMD)

RStudio di Windows menggunakan Command Prompt (cmd.exe) atau PowerShell. Berikut cara mengaktifkan dan menggunakannya untuk Git:

2.1 1.1. Aktifkan Git di RStudio

  1. Buka Tools ‚Üí Global Options ‚Üí Git/SVN

  2. Centang Enable version control interface for RStudio projects

  3. Pastikan path ke git.exe benar, misal:

    C:/Program Files/Git/cmd/git.exe

  4. Klik Apply dan OK.

  5. Restart RStudio.

2.2 1.2. Membuat & Mengelola Proyek Git

  1. File ‚Üí New Project ‚Üí Version Control ‚Üí Git

  2. Masukkan URL repo (GitHub/GitLab), contohnya:

    https://github.com/username/repo-praktikum.git

  3. Pilih folder lokal, klik Create Project.

2.3 1.3. Workflow Dasar di RStudio

Di tab Git (kanan atas):

# di panel terminal RStudio (Cmd/Powershell)
git status
git add nama_file.qmd
git commit -m "Menambahkan panduan"
git push origin main
  • Tombol Commit ‚Üí centang file ‚Üí isi pesan ‚Üí Commit, lalu Push.
  • Gunakan Pull sebelum mulai kerja untuk sinkronisasi.

üîó Referensi resmi RStudio Git setup: https://r-bio.github.io/git-installation/

3 2Ô∏è‚É£ Git di VSCode (Windows + Integrated Terminal)

VSCode memiliki integrasi Git yang powerful dan terminal bawaan (PowerShell by default).

3.1 2.1. Buka & Clone Repo

  1. Jalankan VSCode

  2. Tekan Ctrl+Shift+P ‚Üí ketik Git: Clone

  3. Tempel URL repo, misal:

    https://github.com/username/repo-praktikum.git

  4. Pilih folder lokal, tunggu proses clone selesai.

3.2 2.2. Integrated Terminal

  • Buka terminal dengan `Ctrl+`` (backtick)
  • Default menggunakan PowerShell. Jika ingin Bash:
    • Tekan tombol ‚åÑ di pojok kanan terminal ‚Üí pilih Select Default Profile ‚Üí Git Bash
    • Restart terminal.

3.3 2.3. Workflow Dasar di VSCode

3.3.1 A. Menggunakan GUI Source Control

  1. Klik icon Source Control (ikon percabangan)
  2. Centang perubahan ‚Üí tulis pesan commit ‚Üí ‚úîÔ∏è Commit
  3. Klik ‚Ä¶ ‚Üí Push atau otomatis muncul tombol Push.

3.3.2 B. Menggunakan Terminal

git status
git add .
git commit -m "Memperbarui modul Quarto"
git pull    # tarik perubahan terbaru
git push    # kirim commit lokal

üîó Tutorial resmi Git di VSCode: https://code.visualstudio.com/docs/sourcecontrol/intro-to-git

4 üí° Tips Retensi & Konsistensi

  • Satu perintah berlaku di semua OS: ls, cd, git status, dll.
  • Biasakan GUI + Terminal: GUI memudahkan pemula, terminal menguatkan ingatan sintaks.
  • Selalu pull sebelum push: Hindari konflik merge.
  • Gunakan branch: untuk fitur baru (git checkout -b fitur-XYZ).

Semoga panduan ini membantu kamu menguasai Git secara mulus di kedua IDE. Setelah nyaman, lanjut ke topik Continuous Integration (CI) atau best practices branching!
